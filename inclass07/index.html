The programs use session_start() and then store a boolean and a timestamp in $_SESSION when a correct password is provided. On future requests the page reads $_SESSION['is_authed'] and $_SESSION['authed_at'] to decide whether authentication happened recently. The browser keeps only the PHPSESSID cookie while the real session variables remain on the server. If the session expires or is destroyed, the site no longer considers the user authenticated.
